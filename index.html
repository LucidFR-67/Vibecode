Introduction — What Even Is VibeCoding?

VibeCoding is the art of working with an AI to produce code you could never write yourself. You're not learning to code. You're learning to direct.

Think of yourself as a creative director and the AI as your developer. Your job is to have clear ideas and communicate them well. The AI's job is to translate those ideas into actual CSS.

Bad creative directors give vague instructions and wonder why the result is wrong. Good creative directors give specific direction, review the output critically, and iterate until it's right. That's the entire skill.

Important mindset

The first output the AI gives you is almost never the final result. That's not failure — that's the process. Expect five to fifteen rounds of iteration on anything ambitious. Every round gets you closer.

The designers you see with stunning profiles didn't get it right on prompt one either.

Step 1 — Decide What You Want Before You Open The AI

This is the step most people skip and it's why they get mediocre results. If you open the AI and say "make my profile look cool," you're going to get something generic. The AI has no taste without direction. Your taste becomes its taste, only if you communicate it clearly.

Where to Get Inspiration

  • Pinterest — Search for "dark UI design," "profile page aesthetic," "glassmorphism card," "cyberpunk interface." Save screenshots of whatever feels right.
  • Dribbble and Behance — Professional design portfolios. Search "user profile UI," "character card design," "dark theme app." Very high quality.
  • Games — Video game UIs are a goldmine. Menu screens, character select, loading UIs. Screenshot them.
  • Anime — Opening sequences, title cards, character profile screens. Some of the most distinctive visual styles come from anime aesthetics.
  • Movies — Sci-fi interfaces (Blade Runner, The Matrix, Westworld), fantasy aesthetics, horror minimalism.
  • Canva — Great for rough mood board assembly. Drop your screenshots in, pick color swatches, try font combinations.
  • Handwritten sketches — A phone photo of a rough box diagram you drew on paper is a completely valid reference.

Build Two Separate Plans

Mobile and desktop are completely different experiences and should be designed separately. Your mobile layout is viewed on a small vertical screen — things need to be stacked, text readable at small sizes. Your desktop layout has wide horizontal space — side-by-side columns, richer hover interactions.

Example layout brief 
MOBILE:
- Full-width hero banner with avatar overlapping bottom edge
- Username and stats below the avatar, centered
- Bio text centered, short
- Character cards stack as single column
- Clean, minimal, text is big and readable

DESKTOP:
- Two-column layout: left sidebar has avatar + profile info
- Character cards in 3-column grid on the right
- Sidebar stays fixed while content scrolls
- More visual decoration visible, subtle background texture

Step 2 — Give The AI The Right Context

This is the single biggest difference between getting good results and getting garbage. An AI cannot produce a good CSS override if it doesn't understand the structure of what it's overriding.

Every session with the AI should start by providing three things:

  1. The HTML file of the page
  2. The skill.md file
  3. Any CSS you've already generated in previous sessions

And then explicitly tell the AI to read all of them before doing anything. An AI that hasn't read the HTML will hallucinate class names, invent selectors that don't exist, and produce CSS that does nothing or breaks things.

Opening Instruction Template

Prompt template 
I'm going to provide you with three things. Please read all of them
carefully before writing any code.

1. The HTML source of a JanitorAI profile page (attached)
2. A skill.md file that reverse-engineers the page's CSS architecture,
   stable selectors, and component structure (attached)
3. My current CSS override file (if applicable — pasted below)

Do NOT begin writing CSS until you have confirmed you've studied the HTML
structure, identified the stable pp-* and profile-* selectors from skill.md,
and understood the existing override if one exists.

Confirm when you're ready.
Key rule

The confirmation step forces the AI to actually process everything. If the AI starts generating code immediately without confirming — stop it and ask again.

Step 3 — Tell The AI What It's Allowed To Do

AIs are often cautious by default. They'll make minimal changes if they think you want minimal changes. You need to explicitly give permission for bolder work.

Include a permissions block in your prompt:

Permissions block 
PERMISSIONS:
- You may create new wrapper divs or decorative container elements if needed
- You may add purely decorative elements (::before, ::after pseudo-elements,
  gradient overlays, background patterns)
- You may reorganize visual layout using Flexbox or Grid while keeping
  existing HTML elements functional
- You may replace the existing visual layout completely while preserving
  all interactive functionality
- You may introduce new CSS custom properties (variables) for consistency
- You may redesign individual sections completely
- You may add smooth transitions and hover animations
- You may use advanced CSS techniques (backdrop-filter, clip-path, gradients)

RESTRICTIONS:
- Do NOT change any class names in the HTML
- Do NOT remove interactive elements (buttons, links, inputs)
- Do NOT break the existing navigation functionality
- Do NOT use JavaScript
- Always use pp-* and profile-* selectors as primary targets
- Avoid targeting hashed css-XXXXXXXX classes as primary selectors

Step 4 — Always Specify Your Output Format

If you don't tell the AI how to structure its response, you'll get some mix of explanation and code that's a mess to work with. Always include an output format instruction:

Output format template 
OUTPUT FORMAT:
Please respond with exactly three sections:

[HTML ADDITIONS]
Any new HTML elements. If none needed, write: No HTML changes.

[CSS]
A single complete code block with the FULL CSS override file.
Always the complete current state — not just the changes.

[CHANGES SUMMARY]
A brief bullet-point list of what changed and why.

Request the full CSS every time instead of just the changes — so you can copy-paste the whole thing and replace what you had. Much faster, much less error-prone.

Step 5 — Ask Better Questions (Describe Outcomes, Not Feelings)

This is where most people get stuck. The solution is to describe outcomes, not feelings.

Instead of… Say this
"make it prettier" Increase visual hierarchy between the username and the follower count — make the username 1.5× larger and reduce the follower count to muted text
"make it more professional" Replace with a two-column desktop layout where profile info is a fixed left sidebar (280px) and character cards fill the right
"I don't like the cards" The cards feel too flat — add a subtle drop shadow and increase border-radius to 16px to make them feel more elevated
"make the colors better" Switch to deep navy and gold: background #0a0e1a, accent #c9a84c, white text at 90% opacity
"add some animations" Add a subtle fade-in-upward entrance on page load, and scale 1.02 on character card hover with 300ms ease-out transition

Step 6 — Learn To Iterate

The first draft is not the final draft. After the AI gives you output, don't accept or reject it wholesale. Review it section by section.

What to look for

  • Spacing — Does everything feel too cramped? Too spread out? Breathing room matters.
  • Proportions — Is the avatar too large? Is the sidebar too narrow? Do the cards feel imbalanced?
  • Hierarchy — When you look at the page, is it obvious what's most important? Does your eye know where to go first?
  • Typography — Is text legible? Does size difference feel intentional? Does the font fit the aesthetic?
  • Color — Is there enough contrast? Does the palette feel cohesive? Are accent colors used consistently?
  • Animations — Do they feel smooth? Too fast? Too slow? Do they start in the wrong place?
  • Responsiveness — Does it look good at mobile width? Are there weird breakpoints where things fall apart?

Good iteration feedback examples

Example feedback

"The card hover animation is too fast. The scale increase also feels too large at 1.05. Reduce the scale to 1.02 and slow the transition to 400ms."

Example feedback

"The username heading doesn't feel prominent enough against the background. Increase its font size and add a subtle text shadow to give it more presence."

Never say: "It looks bad." "I don't like it." "Make it better." These tell the AI nothing.

Step 7 — Use Another AI To Improve Your Prompts

This sounds unusual but it works incredibly well. Open a second AI conversation — can be the same model or different. This AI is just a prompt consultant. Don't paste your HTML or CSS there.

Prompt consultant template 
I'm trying to write a prompt for an AI developer to style a profile page.
Here is my current prompt draft. Please review it for:
- Ambiguity (anything that could be misinterpreted)
- Missing information (what would a developer need to know)
- Vague adjectives (replace with specific measurable descriptions)
- Missing references or constraints

Then rewrite the prompt to be cleaner and more precise.

[paste your prompt here]

The rewritten version will often be noticeably better. You'll start to internalize this process and your first drafts will get better over time.

Step 8 — Choosing The Right AI Model

Context window matters

The HTML file for a JanitorAI profile page is large. It contains thousands of lines of HTML and embedded CSS. You're also providing skill.md. Together these are a significant amount of text.

Lightweight or fast models sometimes truncate what they read. They skip sections of the HTML and then produce CSS that doesn't match the actual page structure. This leads to selectors that don't work.

Use a model with a large context window and strong instruction-following.

What to watch for

  • If an AI generates selectors you don't recognize from skill.md, it may be hallucinating class names. Compare against the stable selectors listed in skill.md.
  • If an AI produces extremely short responses when you've provided a lot of context, it may be truncating. Try a different model.
  • If an AI keeps losing track of your instructions between messages, restate your full context at the start of each new message.

Step 9 — The Continuous Improvement Cycle

Good vibecoding is a loop:

  1. Study — Review your inspiration. Know what you're aiming for.
  2. Improve Prompt — Write a clear, specific prompt. Use a second AI to refine it if needed.
  3. Generate — Provide context (HTML, skill.md, existing CSS) and submit.
  4. Review — Look at the output critically. Break it down by section.
  5. Critique — Write specific, measurable feedback.
  6. Repeat — Feed the feedback back as the next prompt.

After five projects you'll find your first prompts are already closer to what you want. After ten you'll be producing results in two or three rounds that used to take fifteen.

Style Archetypes

Use these as starting aesthetic briefs to hand to your AI. Describe outcomes, not just labels.

Dark Luxury / Editorial

Aesthetic brief 
Near-black background (#0a0a0a). Deep charcoal surfaces (#111111).
Single warm platinum accent (#d4c5a9). Serif headings (Playfair Display
or Cormorant Garamond). Sans-serif body (Inter or Geist).
Thin 1px borders at low opacity. No shadows, no glows.
Everything communicates restraint and cost.
Wide letter-spacing on labels. Generous whitespace.
Character cards: tall portrait format, minimal text overlay.

Neon Noir / Cyberpunk

Aesthetic brief 
Very dark blue-black background (#03040a). Electric cyan (#00e5ff) or
hot pink (#ff2d78) accent. Monospaced or tech sans-serif font (IBM Plex Mono,
Space Mono). Bright glowing borders (box-shadow: 0 0 8px var(--accent)).
Scanline texture (repeating-linear-gradient). Uppercase labels.
Grid overlay at very low opacity. Animated blinking or pulse effects.

Baroque Luxury

Aesthetic brief 
Deep jewel tones: sapphire, emerald, ruby on near-black.
Gold accents (#c9a84c, #e8d5a3). Ornamental dividers using Unicode
characters or CSS borders. Heavy use of box-shadow for depth.
Serif typefaces with italic emphasis. Everything feels weighty and important.
Character cards with thick ornate borders and warm background tint.

Troubleshooting

The CSS does nothing

Most common cause: the AI used the wrong selectors. Chakra UI's hashed css-XXXXXXX classes change with every deployment. If the AI targeted those instead of the pp-* and profile-* classes, the CSS won't apply.

Fix

"Your selectors don't seem to be working. Please cross-reference the skill.md selector table and confirm you're using the stable pp-* and profile-* classes. Rewrite the CSS using only those selectors."

The AI keeps forgetting earlier instructions

This happens in long sessions. The AI's attention gets spread thin.

Fix

Start a new chat and reattach all materials. Sometimes a fresh start is faster than trying to fix a confused session.

Something breaks when you apply the CSS

Fix

"When I applied the CSS, [X component] broke. Please review the current CSS and remove or correct any rules that affect [X element] beyond its visual appearance."

The design looks wrong on mobile

Fix

"The desktop layout looks correct. However the mobile layout has these specific issues: [list them]. Please add mobile-specific media query rules to fix these. Desktop styles should remain unchanged."

Quick Reference — Common Tweaks

Color change

Prompt 
Change the accent color from [current] to [new]. Update every place the
current accent appears — borders, hover states, the tab indicator, the
follow button, and any text using the accent color.

Card hover effect

Prompt 
Change the character card hover effect. Remove the scale and replace with:
the card rises upward by 4px using transform: translateY(-4px),
and adds a box-shadow below it using accent color at 20% opacity.

Background change

Prompt 
Change the page background. Instead of a solid dark color, use a very subtle
radial gradient from a slightly lighter version at the center to a darker
version at the edges. Keep the difference very subtle — about 10% contrast.

The Pre-Session Checklist

Before starting any new theming session, run through this list:

  • HTML file attached or ready to paste
  • skill.md attached or ready to paste
  • Previous CSS attached (if continuing from prior session)
  • Mobile layout goals written out specifically
  • Desktop layout goals written out specifically
  • Visual references collected (screenshots, descriptions, or URLs)
  • AI instructed to study all materials before writing any code
  • AI confirmed it has read the materials
  • Permissions block included in prompt
  • Output format specified (HTML block, CSS block, summary)
  • AI reminded to use pp-* and profile-* selectors from skill.md
  • AI instructed to provide FULL CSS file every time (not just changes)
  • Mobile and desktop layouts addressed in the prompt
  • Prompt reviewed for vague adjectives (replaced with specific descriptions)
  • Prompt reviewed for missing measurements (added where possible)

The gap between a mediocre AI-generated theme and a stunning one is almost entirely in how clearly the creative direction was communicated.

The AI has all the technical skill you could need. What it lacks is your taste, your vision, and your specific aesthetic sense. Your job is to transmit those as precisely and completely as possible.

Every round of iteration is a round of getting better at communicating. Don't rush the process. Enjoy it. The conversation with the AI is part of the creative work.
